二、CSRF

该部分参考链接
CSRF（Cross Site Request Forgery, 跨站域请求伪造）是一种网络的攻击方式。CSRF 攻击可以在受害者毫不知情的情况下以受害者名义伪造请求发送给受攻击站点，从而在并未授权的情况下执行在权限保护之下的操作。
CSRF的几种防御策略如下：

2.1 验证 HTTP Referer 字段

通过验证HTTP请求中的referer字段，可以判断该请求是否来自合法的页面，比如银行可以判断HTTP请求是否来自本银行的页面，如果referer字段是不合法的页面，则该请求有可能是黑客的CSRF攻击。
该种方法的缺点是，黑客可以使用其他的手段来篡改referer字段的值。即使黑客无法篡改referer的值，某些用户考虑到自身的隐私问题会将浏览器中的发送referer的选项关闭，这样网站就无法根据referer字段判断是否是合法请求。

2.2 在请求地址中添加 token 并验证

CSRF 攻击之所以能够成功，是因为黑客可以完全伪造用户的请求，该请求中所有的用户验证信息都是存在于 cookie 中，因此黑客可以在不知道这些验证信息的情况下直接利用用户自己的 cookie 来通过安全验证。要抵御 CSRF，关键在于在请中放入黑客所不能伪造的信息，并且该信息不存在于 cookie 之中。可以在 HTTP 请求中以参数的形式加入一个随机产生的 token，并在服务器端建立一个拦截器来验证这个 token，如果请求中没有 token 或者 token 内容不正确，则认为可能是 CSRF 攻击而拒绝该请求。
这种方法要比检查 Referer 要安全一些，token 可以在用户登陆后产生并放于 session 之中，然后在每次请求时把 token 从 session 中拿出，与请求中的 token 进行比对，但这种方法的难点在于如何把 token 以参数的形式加入请求。对于 GET 请求，token 将附在请求地址之后，这样 URL 就变成http://url?csrftoken=tokenvalue，而对于 POST 请求来说，要在 form 的最后加上<input type=”hidden” name=”csrftoken” value=”tokenvalue”/>这样就把 token以参数的形式加入请求了。但是，在一个网站中，可以接受请求的地方非常多，要对于每一个请求都加上token是很麻烦的，并且很容易漏掉，通常使用的方法就是在每次页面加载时，使用 javascript 遍历整个 dom 树，对于 dom 中所有的 a 和 form 标签后加入 token。这样可以解决大部分的请求，但是对于在页面加载之后动态生成的 html 代码，这种方法就没有作用，还需要程序员在编码时手动添加 token。
该方法还有一个缺点是难以保证 token 本身的安全。特别是在一些论坛之类支持用户自己发表内容的网站，黑客可以在上面发布自己个人网站地址。由于系统也会在这个地址后面加上 token，黑客可以在自己的网站上得到这个token，并马上就可以发动CSRF攻击。为了避免这一点，系统可以在添加token的时候增加一个判断，如果这个链接是链到自己本站的，就在后面添加 token，如果是通向外网则不加。不过，即使这个 csrftoken 不以参数的形式附加在请求之中，黑客的网站也同样可以通过 Referer 来得到这个 token 值以发动 CSRF 攻击。这也是一些用户喜欢手动关闭浏览器 Referer 功能的原因。

2.3 在 HTTP 头中自定义属性并验证

这种方法也是使用 token 并进行验证，和上一种方法不同的是，这里并不是把 token 以参数的形式置于 HTTP 请求之中，而是把它放到 HTTP 头中自定义的属性里。通过 XMLHttpRequest 这个类，可以一次性给所有该类请求加上 csrftoken 这个 HTTP 头属性，并把 token 值放入其中。这样解决了上种方法在请求中加入 token 的不便，同时，通过 XMLHttpRequest 请求的地址不会被记录到浏览器的地址栏，也不用担心 token 会透过 Referer 泄露到其他网站中去。
然而这种方法的局限性非常大。XMLHttpRequest 请求通常用于 Ajax 方法中对于页面局部的异步刷新，并非所有的请求都适合用这个类来发起，而且通过该类请求得到的页面不能被浏览器所记录下，从而进行前进，后退，刷新，收藏等操作，给用户带来不便。另外，对于没有进行 CSRF 防护的遗留系统来说，要采用这种方法来进行防护，要把所有请求都改为 XMLHttpRequest 请求，这样几乎是要重写整个网站，这代价无疑是不能接受的。

mx master 1.0 使用感受

Posted on 2017-08-19 | Edited on 2017-09-05 | In Play | Comments:

前言

自从今年年初换了2016款的rmbp，外设也跟着大换血。早就知道mac的触控板是所有笔电中最好用的，经过这段时间的使用，确实如此，触摸手感一流，手势控制丝般顺滑，有时候确实是想扔下鼠标，只用触控板，码字的时候，手部不需要很大的移动就能操控鼠标。

但是问题来了，在需要用ps或者AE的时候触控板便显得很无力，于是就苦苦寻找一款适合2016款rmbp的鼠标，由于目前市面上还没有雷电3接口的有线鼠标，所以蓝牙鼠标就显得更优雅一点，苹果官方鼠标手感欠佳，最后选择了罗技mx master 1.0 这款鼠标，符合大众的使用习惯。

我使用这款鼠标也有大半年了，目前mx系列的鼠标已经升级至2.0，所以本文可能不具参考价值。这次是应一位评测界的资深爱好者的要求来写这篇文章，也是分享我自己使用该款鼠标的感受。

外观&手感

优点

  • 作为办公鼠标，不得不说这款鼠标真的很大，手小的可以选择它的兄弟产品：mx anywhere。
  • 对于我自己来说，这款鼠标手感很棒，外形符合人体工程学。类肤材质摸起来很高档，对于手汗症的我来说，再也有没有鼠标脏湿漉漉的情况了。
  • 鼠标的手势键设置在大拇指下方，用来替代触控的手势，使用起来也很方便。

缺点

  • 鼠标尾部微微翘起，和其他鼠标不太一样，刚开始可能会不太适应，稍稍有点累，但时间 长了可能会习惯。

按键

  • 这款鼠标的按键包括：左右键，双向按键滚轮，滚轮模式切换按键，两个侧键，一个横向滚轮，一个手势按键，底部分别是鼠标开关，蓝牙连接按键，设备切换按键。
  • 忍不住吐槽罗技的微动，手感真的渣，严重影响整个鼠标的使用，也可能是我之前用G502手感太好，感觉这微动撑不过1年。
  • 网络上已经有很多人的滚轮回滚，我的其实也有略微回滚，习惯了之前的机械滚轮，一时间难以适应。自适应滚轮再从自由滚动模式切换到分段滚动模式的时候会有『咔』的一声，并且可能会发生回滚，在浏览网页的时候尤其明显，个人并不觉得这个设计很好用。
  • 在安装了罗技的驱动程序后，手势按键可以自由定义，基本能代替触摸板的手势。
  • 另外一个横向滚轮，这个也比较少见，在视频编辑软件或者excel中使用频率较高，当然也可以自定义功能。但是我的这个滚轮手感并不好，滚动模式是自由滚动，但是页面确实分段滚动，不知道是不是个例。

连接&电量

  • 支持unifying优联连接和蓝牙连接，可以记忆三台设备，通过底部的按钮切换。
  • 在2016rmbp上使用蓝牙连接并不稳定，鼠标指针经常会出现『飘』的情况，使用优连情况会稍好，但也仅仅是减少这种情况。
  • 附送了充电的micro usb充电线，可惜不支持有线连接。
  • 4，5年前用过无线鼠标，大概2，3周就要换一次电池，mx master的电量对于我来说是很耐用的，正常使用至少3周，一般一个月左右充一次电。

总结

mx master 1.0 总得来说让我很失望，花500多买的行货用起来甚至不如200多海淘的G502，最烦的是鼠标经常飘，蓝牙模式下很严重，在wifi较多的办公环境中几乎没法用，这点也可能是蓝牙鼠标的通病吧。其次是鼠标dpi最大为1600，新版的master已经提升到了4000dpi。

下一步打算找个magic mouse体验一下。

以上内容仅代表个人观点。

目前为止自己做的板子已经能稳定运行起来了，基本和官网的效果一致。总结以下几个关键点。

  1. tag和anchor要配置成功，tag这块需要在编译文件中写入一条预编译命令。anchor正常配置，编号从1开始（目前官方正在修复，以使得从0开始，从而和ros中一致）
  2. anchor的位置选择，尽量有差异性，xyz不同坐标上都要形成有效的定位。最好在空旷一点的地方，一个大的仓库或者走廊中。
  3. 调试中有条件的可以加保护网，虽然我没有加，摔了几次。
  4. 多看官方的文档，多读程序。

后期准备加入TDMA的双四旋翼和TDOA的多四旋翼，有时间再做几个crazyflie吧。希望相关领域的小伙伴们多多交流。新年快乐。
