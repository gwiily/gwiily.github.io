在模拟器中安装google play版部落冲突

Posted on 2016-05-15 | Edited on 2017-09-05 | In Play | Comments:

想在模拟器中玩部落冲突，但是在google市场里无法安装部落冲突以及皇室战争，显示“您的设备不兼容”，只需要修改一个文件即可。步骤如下：

  1. 首先模拟器必须自带root或者可以root，安装re管理器
  2. 进入re管理器，./system目录，找到build.prop，长按选择以文本编辑的方式打开
  3. 找到ro.product.model=xxx,修改xxx为手机型号，如GT-i9100,等，以及其他参数也修改一下，如name
  4. 修改完成后右上角选择保存并退出，重启模拟器即可

./ | 根目录，包含Makefile

+ init         | 主函数
+ config       | 配置文件 
+ drivers      | 硬件驱动文件
|  + src       | 驱动文件源代码 
|  + interface | 驱动文件头文件. Interface to the HAL layer
+ hal          | 硬件抽象层 
|  + src       | HAL 源代码
|  + interface | HAL 头文件及接口
+ modules      | 固件操作源码和头文件
|  + src       | 固件子函数
|  + interface | 头文件. 配置固件环境
+ utils        | 工具包代码. 实现诸如控制台的程序.
|  + src       | 工具包源码
|  + interface | 工具包头文件及接口
+ platform     | 固件的特殊文件，不一定被使用
+ tools        | 杂项. scripts for LD, OpenOCD, make, version control, ...
|              | *** 这两个以下文件夹中包含未修改文件 ***
+ lib          | 库文件
|  + FreeRTOS  | FreeRTOS 源码， 删除了不必要的问题
|  + STM32...  | STM32的外围设备的库文件
|  + CMSIS     | 核心抽象层

./ | Root, contains the Makefile

+ init         | Contains the main.c
+ config       | Configuration files
+ drivers      | Hardware driver layer
|  + src       | Drivers source code
|  + interface | Drivers header files. Interface to the HAL layer
+ hal          | Hardware abstaction layer
|  + src       | HAL source code
|  + interface | HAL header files. Interface with the other parts of the program
+ modules      | Firmware operating code and headers
|  + src       | Firmware tasks source code and main.c
|  + interface | Operating headers. Configure the firmware environement
+ utils        | Utils code. Implement utility block like the console.
|  + src       | Utils source code
|  + interface | Utils header files. Interface with the other parts of the program
+ platform     | Platform specific files. Not really used yet
+ tools        | Misc. scripts for LD, OpenOCD, make, version control, ...
|              | *** The two following folders contains the unmodified files ***
+ lib          | Libraries 
|  + FreeRTOS  | Source FreeRTOS folder. Cleaned up from the useless files 
|  + STM32...  | Library folders of the ST STM32 peripheral libs 
|  + CMSIS     | Core abstraction layer

Crazyflie 2.0 系统结构

Posted on 2015-12-02 | Edited on 2017-09-05 | In Quadcopter , Crazyflie 2.0 | Comments:

翻译：https://wiki.bitcraze.io/projects:crazyflie2:architecture:index

Crazyflie 2.0 硬件中主要包含两个微控制器：

  • 一个NRF51，Cortex-M0，主要管理无线通信和电源管理：

    • 逻辑开关
    • 给系统的其他部分供电（STM32,传感器和扩展板）
    • 电源充电管理和电压测量
    • Master radio bootloader（无线电主引导）
    • 无线电和蓝牙连接
    • 检测和检查已安装的扩展版

  • 一个STM32F405，Cortex-M4@160MHz，负责主要的任务和飞行控制以及其他任务：

    • 传感器数据读取和电机控制
    • 飞行控制
    • 遥测（包括电池电压）
    • 其他用户开发的

下面这张图解释了crazyflie是如何工作的：

nRF51822

nRF51的主要任务是负责无线电通信和电源管理，它其实是作为一个无限网桥（它将通信的原始数据发送给STM）

Crazyflie 2.0 使用的无线电为CRTP和蓝牙，但是硬件还支持一些其他协议，例如ANT。CRTP模式和Crazyradio USB适配器兼容，并提供了2Mbit/s的数据链路。测试表明该无线电路的延迟是360us和1.26ms之间，在2Mbps的速率下可以传输1到32字节的无重复数据。使用蓝牙传输的最小延迟为7.5ms，目前执行的延迟为20ms。使用CRTP和Crazyradio传输的好处是它很容在支持USB 的书记中使用，使他成为首选的破解和实验使用的小型四旋翼。蓝牙传输可以在大多数移动设备上实现。

nRF51芯片的另一个特殊性是，他是设计为使用纽扣电池供电的芯片，这意味着它可以很好的适用于低能量操作，所以它还负责电源管理。它处理着开/关逻辑，这意味着NRF51总是处于通电状态，当你长时间按下开/关按钮的时候可能会有不同的作用（这是用于启动引导程序的）。另外，它也可以允许从一个扩展口的pin针上唤醒一个外部电源。

STM32F405

该SMT32运行了主要固件。即使他是被NRF51引导，但它仍然可以控制NRF51。它实现了飞行控制和所有的通信算法。扩展端口主要连接在STM32上，扩展驱动板也是被连接在STM32上的。

STM32F405有196kb的内存，这对所有开发者来说应该是足够的（著名的最后一句话…）。这对于飞行控制来说是绰绰有余的，但是它允许了更多的计算密集型的算法，比如，惯性传感器和GPS传感器数据的融合。

内部MCU通信

两个CPU之间的通信时通过符号连接协议处理，它是一个简单的基于分组的协议，这是我们提出的一种可扩展的通信方案。

syslink提供了两颗CPU之间所有的请求信息，STM32是主CPU，而NRF51是从CPU，我们尽可能的让NRF51处理一些简单的任务，而STM32处理复杂的算法。

SYSLINK消息举例：

  • 初始无限信号的发送和接受
  • 电源管理和测量

crazyflie开发入门

Posted on 2015-12-01 | Edited on 2017-09-05 | In Quadcopter , Crazyflie 2.0 | Comments:

翻译：https://wiki.bitcraze.io/doc:crazyflie:dev:starting

前期准备：

开发前，首先你需要一台crazyflie（1.0或者2.0）和crazyradio（PA）。如果你需要做复杂的开发工作，你还需要JTAG和SWD编程电缆，

本页面假设你使用了 bitcraze VM 0.6，它集成了所有的开发工具，对于开发者来说是很方便的，VirtualBox需要1G运存运行在windows，linux和mac上面。或者你可以自己搭建开发环境。

crazyflie是使用C语言编写的，这意味着你需要掌握C语言的编程技术，最好要有嵌入式开发的经验。记住一点，Crazyflie1.0的RAM为20K，Crazyflie2.0的为196K。

官方的主机端和客户端都是使用Python语言编写的，所以你需要掌握Python语言的一些知识，在网上有很多学习资料。

第三方库和客户端还存在有其他语言，尽管如此，你还是更容易从Python语言中获得一些东西，因为它是与固件更新。

所有的源代码都被托管在github上，你需要熟悉git的使用方法，并贡献你的修改，这样可以减少你的工作量。

系统结构：

编译和刷新固件

STM32

从github上克隆crazyflie固件工程或者在虚拟机中使用“更新所有项目”脚本更新固件。crazyflie2.0切换至crazyflie2分支，crazyflie1切换至主分支。

执行以下命令

~$ cd projects/crazyflie-firmware/

crazyflie-firmware$ git checkout master # For crazyflie 1

crazyflie-firmware$ git checkout crazyflie2 # For Crazyflie 2.0

然后执行：

crazyflie-firmware$ make

crazyflie-firmware$ make cload

注意编程电缆

为了使用radio刷写程序和调试，我们需要JTAG或者SWD电缆。

  • crazyflie1需要JTAG电缆
  • crazyflie2.0需要SWD/SWV电缆
    默认情况下可以使用openODC支持的电缆。Bitcraze团队使用的是STlink-V2。其他电缆比如JLink也可以使用，但是它需要手动输入刷写命令和修改Eclipse配置。

Makefile文件默认配置为STlink-V2电缆，如果你要配置另外一根编程电缆，在项目根目录下创建一个.mk文件。文件内容如下：

OPENOCD_INTERFACE ?= interface/stlink-v2.cfg

OPENOCD_TARGET ?= target/stm32f4x_stlink.cfg

crazyflie2.0介绍

Posted on 2015-11-29 | Edited on 2017-09-05 | In Quadcopter , Crazyflie 2.0 | Comments:

Crazyflie2.0是一个多功能的飞速发展平台，它的重量只有27克，适合在你的手掌。其先进的功能，使得它非常适合开发人员和蓝牙低功耗（LE）功能，可以很容易地从移动设备飞行。由于其体积小，重量轻，这是非常适合室内使用，但也可以很容易上面盘旋你的房子，因为它可以在你的餐桌上徘徊。配件设计为无需焊接，只通过简单的连接到电机的电路板，就可以准备起飞了。

功能：

  • Durable design。耐用的设计。
  • Easy assembly and no soldering required。简单的组装并不需要焊接。
  • Supports expansion boards with automatic detection。支持扩展卡。
  • Supports flying from iOS and Android with Bluetooth LE support as well as from Windows/MacOSX/Linux with the Crazyradio or Crazyradio PA。支持IOS,Android,蓝牙控制，以及使用Crazyradio在Windows/MacOSX/Linux上控制。
  • Tested to above 1 km radio range line-of-sight (LOS) with Crazyradio PA。Crazyradio PA可以在1km范围内控制。
  • Wireless firmware update。无线固件升级。
  • On-board charging via standard uUSB。USB充电设计。
  • Dual-MCU architecture with dedicated radio/power management SoC for advanced applications.双MCU设计，单独的电源管理芯片。

  • Real-time logging, graphing, and setting of variables as well as full access to the available expansion boards with use of a Crazyradio or Crazyradio PA and a personal computer。实时记录日志，绘图，变量的设置。
    crazyflie2.0的硬件规格

  • Mechanical specification
    • Weight: 27g
    • Size (WxHxD): 92x92x29mm (motor-to-motor and including motor mount feet)
  • Radio specification
    • 20 dBm radio amplifier tested to > 1 km range LOS with Crazyradio PA
    • Bluetooth Low Energy support with iOS and Android clients available (tested on iOS 7.1+ and Android 4.4+)
    • Radio backwards compatible with original Crazyflie and Crazyradio
  • Micro-controllers
    • STM32F405 main application MCU (Cortex-M4, 168MHz, 192kb SRAM, 1Mb flash)
    • nRF51822 radio and power management MCU (Cortex-M0, 32Mhz, 16kb SRAM, 128kb flash)
  • uUSB connector
    • On-board LiPo charger with 100mA, 500mA and 980mA modes available
    • Full speed USB device interface
    • Partial USB OTG capability (USB OTG present but no 5V output)
  • IMU
    • 3 axis gyro (MPU-9250)
    • 3 axis accelerometer (MPU-9250)
    • 3 axis magnetometer (MPU-9250)
    • High precision pressure sensor (LPS25H)
  • Flight specification
    • Flight time with stock battery: 7 minutes
    • Chraging time with stock battery: 40 minutes
    • Max recommended payload weight: 15 g
  • Expansion connector with
    • VCC (3.0V, max 100mA)
    • GND
    • VCOM (unregulated VBAT or VUSB, max 1A)
    • VUSB (both for input and output)
    • I2C (400kHz)
    • SPI
    • 2 x UART
    • 4 x GPIO/CS for SPI
    • 1-wire bus for expansion identification
    • 2 x GPIO connected to nRF51

  • 8KB EEPROM

    翻译：https://wiki.bitcraze.io/projects:crazyflie2:index

pixhawk在linux（ubuntu14.04）下的开发环境搭建及源码编译

Posted on 2015-11-25 | Edited on 2017-09-05 | In Quadcopter , PIXHAWK开发 | Comments:

参考：http://dev.px4.io/starting-installing-linux.html

我使用的是UBUNTU14.04，安装了win10+UBUNTU的双系统，方便在各个系统下的调试开发。

安装过程官方文档已经说的比较清楚了，安装的时候最好不要挂代理，否则可能会无法下载某些软件和源码。

（本文仅针对硬件为PIXHAWK的开发环境搭建，其他硬件请参考官方文档）

第一步：执行以下三条命令

sudo add-apt-repository ppa:george-edison55/cmake-3.x -y
sudo apt-get update
sudo apt-get install python-argparse git-core wget zip \
python-empy qtcreator cmake build-essential -y

第二步：执行以下命令删除串口模式管理器

sudo apt-get remove modemmanager

第三步：执行以下命令更新依赖包：

sudo add-apt-repository ppa:terry.guo/gcc-arm-embedded -y
sudo apt-get update
sudo apt-get install python-serial openocd \
flex bison libncurses5-dev autoconf texinfo build-essential \
libftdi-dev libtool zlib1g-dev genromfs \
python-empy

第四步：执行以下命令安装相应版本的GCC

sudo apt-get install gcc-arm-none-eabi=4.8.3-18ubuntu2+12 -y
（对于老版本的ubuntu，例如Ubuntu 14.04 (Trusty) and 14.10，执行以下命令）
sudo apt-get install gcc-arm-none-eabi -y

第五步：更该用户权限，将当前用户添加进组中$USER替换为当前用户：

sudo usermod -a -G dialout $USER

第六步：执行以下命令下载PX4源码

mkdir -p ~/src
cd ~/src
git clone https://github.com/PX4/Firmware.git

第七步：编译源码

cd Firmware
make px4fmu-v2_default

显示以下为编译成功：

`

第八步：上传固件至PIXHAWK

make px4fmu-v2_default upload
